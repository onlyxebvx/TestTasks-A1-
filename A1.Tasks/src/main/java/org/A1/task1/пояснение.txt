Так как в условии 1-го задания явно указано решение через тип данных int32,
а в примере есть число (2149583360)  выходящее за пределы этого типа данных. Я привел второе решение данной задачи,
в котором тип int заменил на тип данных long.
Что позволило нам не только передавать в функцию в качестве параметра
числа превышающие диапозон -2147483648 до 2147483647(то есть, наше число 2149583360).
Также тип long использует беззнаковое представление чисел, и не будет возвращать отрицательные значения.
